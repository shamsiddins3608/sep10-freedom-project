# Entry 6: MVP Completion

## Introduction

For this Freedom Project, my topic is **Construction** and my LOYO tool is **A-Frame**. My goal is to create a website that teaches high school students about how technology is used in construction today and how it may be used in the future. This entry focuses on the creation of my **Minimum Viable Product (MVP)**, the challenges I faced, and what I learned throughout the process.

## My MVP

My MVP includes all of the main components needed for a functional website. I created:

* A navigation bar that directs users to different sections of the website.
* Text sections that explain information about construction technology.
* **Part A**, which discusses how technology is currently used in construction.
* **Part B**, which explores how technology may be used in construction in the future.
* An interactive **A-Frame model** that allows users to view a 3D construction-related object.
* A footer at the bottom of the website.

The MVP is functional and allows users to learn about construction technology while interacting with a 3D model.

## How I Made My MVP

To build my MVP, I first created the structure of my website using HTML. After that, I used CSS to style the page and make it easier to navigate. Once the main sections were complete, I focused on implementing my A-Frame model.

The A-Frame portion required the most work because I had to learn new syntax and understand how 3D scenes, entities, cameras, and assets work together. After multiple tests and revisions, I was able to successfully display a 3D model on my website. This feature helps make my website more interactive and engaging for users.

### Evidence of Learning

Example of A-Frame code:

```html
<a-scene>
    <a-entity
        gltf-model="#model"
        position="0 0 -3"
        scale="1 1 1">
    </a-entity>
</a-scene>
```

This code helped me display a 3D model within my website. Learning how each attribute affects the model's appearance was an important part of developing my MVP.

## Challenges and Takeaways

The biggest challenge I faced was working with **A-Frame**. Since it was a new tool and that i switched my tool up mid semester, it took time to understand how to position models correctly and make them appear properly on the page. Sometimes the model would not load, and other times it would appear in the wrong location.

To learn how my tool worked, I researched documentation, experimented with different values, and tested my website frequently. Through this process, I became more comfortable working with new technologies independently.

One important takeaway is that debugging is a normal part of web development. Even when something does not work immediately, testing different solutions, patience, and researching documentation can eventually lead to success.
